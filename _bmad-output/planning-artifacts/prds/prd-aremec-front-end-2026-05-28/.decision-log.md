# Decision Log — AREMEC Clinical Portal PRD

## Session: 2026-05-28

### D-001 — PRD Scope
**Decision**: Scope the PRD to the frontend web application only; excludes VR game, backend, and ML service internals.
**Rationale**: User confirmed focus is on guiding remaining frontend development.

### D-002 — Stakes Level
**Decision**: Thesis-level depth (5-8 pages), moderate rigor on NFRs and compliance.
**Rationale**: Project is an academic thesis with controlled INCN deployment, not a commercial launch.

### D-003 — Working Mode
**Decision**: Fast path selected.
**Rationale**: Rich backlog provided as primary input; user confirmed fast path.

### D-004 — Form Factor & Stack
**Decision**: Desktop-only (1280px+), Spanish UI, React + TypeScript + Tailwind CSS.
**Rationale**: Confirmed by user — clinical consultation room, fixed workstation.

### D-005 — VR Streaming flagged as phase-blocker (OQ-1)
**Decision**: HU-15 live VR view marked as Open Question and phase-blocker.
**Rationale**: Backend documentation only describes a telemetry WebSocket; video streaming mechanism is undocumented and must be resolved before the monitoring screen can be implemented.

### D-006 — Clinical override endpoint flagged (OQ-2)
**Decision**: HU-12 Scenario 2 (manual difficulty override) marked as Open Question.
**Rationale**: No PATCH/PUT endpoint for difficulty override appears in the backend API documentation.

### D-014 — PRD finalized
**Decision**: PRD set to `status: final` on 2026-05-28.
**Notes**: All open questions resolved or explicitly deferred. No unresolved `[ASSUMPTION]` tags. Section 10 (Assumptions) added. OQ-1 (router/state) deferred to architecture phase — not a PRD-level concern.

### D-010 — Stream degradation: auto-reconnect loop, no UI fallback (OQ-1 resolved)
**Decision**: When Cloudflare Stream is unavailable or latency exceeds 2 seconds, the player auto-reconnects in a loop with no alternative UI. The neurologist sees no different view during reconnection.
**Rationale**: Confirmed by user 2026-05-28.

### D-011 — Router/state management deferred to architecture phase (OQ-2 → OQ-1)
**Decision**: Technology choice for routing and state management is not decided at PRD stage; deferred to architecture.
**Rationale**: User confirmed 2026-05-28.

### D-012 — No formal Figma design system; Tailwind from scratch (OQ-3 resolved)
**Decision**: Figma contains screen designs but no formal design system (no tokens, no reusable components). Frontend implements UI from scratch using Tailwind CSS guided by the Figma screens.
**Rationale**: Confirmed by user 2026-05-28.

### D-013 — Inactivity warning out of scope; only 401 handling (OQ-4 resolved)
**Decision**: The 2-minute prior warning (HU-02) is out of scope. Frontend only reacts to 401 responses. Inactivity enforcement is server-side via cookie TTL.
**Rationale**: Confirmed by user 2026-05-28.

### D-008 — VR streaming: Cloudflare Stream + Meta Quest 3S (OQ-1 partially resolved)
**Decision**: Live VR view uses Cloudflare Stream; source is Meta Quest 3S headset; frontend embeds the Cloudflare Stream player.
**Rationale**: Confirmed by user 2026-05-28. Degradation fallback remains open (OQ-1 updated, no longer a phase-blocker for identifying the mechanism, but behavior-on-failure still TBD).

### D-009 — Clinical difficulty override out of scope
**Decision**: Manual difficulty override by neurologist (HU-12 Scenario 2) will not be implemented. Recommendation display is read-only.
**Rationale**: User confirmed 2026-05-28. FR-5.3 updated; OQ-2 removed.

### D-007 — ML null fields treated as "pending" state
**Decision**: `sps_class` and `recommendation` null values render a loading/pending indicator, not an error.
**Rationale**: Backend explicitly states these fields may be null on REST response while ML service processes asynchronously; the WebSocket event carries resolved values.
